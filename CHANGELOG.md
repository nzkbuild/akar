## v0.44.0 — 2026-07-08
Manual-CLI Burden Reduction Design. Audit/design release only — no runtime behavior changes, no src/ modifications, no feature additions. Created a comprehensive 20-section design report (`docs/audits/AKAR_V0_44_MANUAL_CLI_BURDEN_REDUCTION_DESIGN.md`) that defines how AKAR can reduce manual CLI burden (7+ commands per task cycle) and deliver guidance to AI sessions without user copy/paste, without auto-executing project code, without model/API calls, and without modifying Claude Code settings automatically. Key findings: (1) the current 7-command per-task cycle is a manual-CLI trap contradicting the North Star mandate; (2) command consolidation via `akar prepare` and `akar finish` can reduce the cycle to 2 commands by composing existing advisory operations; (3) a managed CLAUDE.md snippet is the smallest, highest-leverage mechanism for AI-facing context delivery — it instructs Claude to read `.akar/NEXT_RUN.md` automatically at session start without auto-execution; (4) seven AI-facing delivery mechanisms evaluated (A–G), with CLAUDE.md snippet accepted as primary and manual status quo kept as fallback; (5) shell wrapper/launcher, file watcher/daemon, and auto-commit are explicitly rejected; (6) all current commands classified for consolidation into prepare/finish groupings; (7) a three-tier AI context size budget designed (Tiny 7 lines, Normal 30, Full current) with token-waste reduction rationale; (8) explicit auto-invocation vs auto-execution distinction with clear safety boundaries; (9) toggle model designed via `.akar/config.toml` + `akar enable`/`akar disable` with all auto features defaulting to OFF; (10) implementation sequenced across 6 future releases (v0.45–v0.50) with strict evidence-before-feature ordering; (11) recommended next release is v0.45.0 Burden Measurement Dogfood to collect baseline data before any implementation. Cargo.toml bumped to v0.44.0 after verification passed.
